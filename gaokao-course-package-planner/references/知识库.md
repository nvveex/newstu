# 知识库

## 1. 常见工作簿模式

### 无分层模式

常见工作表：
- `选考-源数据`
- `教师列表`

`选考-源数据` 常见字段：
- `姓名`
- `学工号`
- `行政班`
- `选考科1`
- `选考科2`
- `选考科3`

适用场景：
- 只有六选三组合，没有现成 `ABC/123` 分层
- 需要先按教师课程班数量拆班，再分成课位包

### 成绩分层模式

常见工作表：
- `选考-源数据`
- `合格考-源数据`
- `教师列表`

`选考-源数据` 常见字段：
- `姓名`
- `学工号`
- `行政班`
- `新科目组合`
- `物理`
- `化学`
- `生物`
- `历史`
- `地理`
- `政治`

`合格考-源数据` 常见字段：
- `姓名`
- `学工号`
- `行政班`
- `物理`
- `化学`
- `生物`
- `历史`
- `地理`
- `政治`

适用场景：
- 用户已经先按成绩把选考分成 `A/B/C`
- 合格考分成 `1/2/3`
- 需要尽量保留原分层，再做课位打包

## 2. 课程班命名标准

### 选考

常见形式：
- `物理选考A`
- `化学选考B`
- `生物选考A`
- `历史选考B`
- `地理选考C`
- `政治选考A`

也可能出现：
- `物理选考A班`
- `物理选考B班`
- `物理选考C班`

### 合格考

常见形式：
- `物理合格考1`
- `化学合格考2`
- `生物合格考3`
- `历史合格考1`
- `政治合格考2`
- `地理合考１`
- `地理合考２`

标准化时常用规则：
- 全角数字转半角
- 全角字母转半角
- `合考` 统一成 `合格考`
- 只在需要与教师表匹配时补 `班` 后缀

## 3. 学生层面的基础逻辑

1. 每个学生共有 `6` 门非语数英科目需要覆盖。
2. 选考的 `3` 门来自 `新科目组合` 或 `选考科1/2/3`。
3. 合格考为剩余未选的 `3` 门。
4. 在成绩分层模式中，优先用 `新科目组合` 判断某科属于选考还是合格考。
5. 不要因为另一张表里误填了值，就改变该科的本来类型。

## 4. 教师表常见情况

教师表一般只有两列：
- `科目`
- `教师`

常见问题：
- 同一课程班完全重复出现两次
- 同一老师带多个课程班
- 课程班命名与学生表略有差异，例如是否带 `班`

默认处理方法：
- 先判断是否是重复录入
- 若用户确认是重复录入，则按 `课程班 + 教师` 去重

## 5. 常见异常

### 学生异常

- 某个学生某科在两张表里都为空
- `新科目组合` 与各科实际分层列不一致
- 合格考列里出现了本应属于选考的课程
- 选考列里出现了本应属于合格考的课程
- 同名学生存在，需要依赖 `学工号`

### 命名异常

- `地理合考１` / `地理合格考1班` 混用
- `物理选考A` / `物理选考A班` 混用
- 单元格只写了 `生物` 这种无法识别层次的值

### 处理原则

1. 先识别是否属于“可自动标准化”的问题。
2. 若标准化后仍无法判断，就列入待确认问题。
3. 如果用户要求忽略异常学生，要在结果里保留说明。

## 6. 常见输出表

详细版常见工作表：
- `选科分层结果`
- `学生原始新分层对比`
- `分层变动统计`
- `不能满足名单`
- `每课位课程清单`
- `教师课位表`
- `说明`

教师简洁版常见工作表：
- `课位包总表`
- `学生课位一览`
- `课程班名单`
- `教师课位一览`
