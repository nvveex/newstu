# 工作流

## 一、执行顺序

### 第 1 步：确认任务口径

先确认用户到底要哪一种结果：
- 只做课程班打包
- 做分层后再打包
- 做完整比较表
- 只要教师简洁版

如果用户已经给出明确口径，直接按口径执行，不重复追问。

### 第 2 步：读取并预检 Excel

至少检查：
- 工作表数量和名称
- 学生总人数
- 每张学生表行数是否一致
- 关键字段是否存在
- 教师表是否有重复课程班

### 第 3 步：汇总不确定项

把所有不确定问题一次性列出来让用户确认，例如：
- 哪些教师表重复行只是重复录入
- 某个异常学生怎么处理
- 优先保哪些科目和分层
- 班额上下限是多少

确认后再正式求解。

### 第 4 步：标准化数据

统一：
- 课程班命名
- 全角/半角字符
- 是否带 `班`
- 组合字符串中的分隔符

### 第 5 步：建立课程班集合

按教师表确定实际可开的课程班。

常见做法：
1. 读取教师表课程班
2. 去重
3. 建立 `科目 + 类型 + 层次 -> 课程班/教师` 映射

### 第 6 步：建立学生目标课程集合

对每个学生，明确这 `6` 门课分别是什么：
- 哪些是选考
- 哪些是合格考
- 原始分层是什么
- 新分层要分到哪里

### 第 7 步：建立约束并求解

至少包含：
- 学生不能同一课位上两门课
- 每个学生在每个课位包里恰好一门课
- 每个课程班只能属于一个课位包
- 教师同课位不能冲突
- 班额满足上下限

如果是成绩分层模式，再加入：
- 尽量保留原始分层
- 指定科目优先级
- A 层控制
- B/C 结构约束

### 第 8 步：解后复核

至少复核：
- 每个课位包课程班数量
- 每个课位包学生总量是否等于总人数
- 教师是否冲突
- 是否有学生缺课或重复占包
- 未满足名单和原因是否完整

### 第 9 步：导出结果

输出详细版和教师简洁版。

## 二、推荐输出结构

### 详细版

建议包含：

#### 1. 选科分层结果

按课位列出选考课程班，包含：
- 课位包
- 课程班
- 教师
- 学生数
- 原始层次构成
- 学生名单

#### 2. 学生原始新分层对比

按学生逐科列出：
- 原始分层
- 新分层
- 是否变动
- 所属课位包

#### 3. 分层变动统计

按科目、类型、层次统计：
- 原始人数
- 新人数
- 原层保留人数
- 调出人数
- 调入人数

#### 4. 不能满足名单

列出：
- 学生
- 科目
- 类型
- 原始分层
- 新分层
- 新课位包
- 原因

#### 5. 每课位课程清单

按课位包列出所有课程班，包含：
- 课程班
- 科目
- 类型
- 教师
- 学生数
- 学生名单

#### 6. 教师课位表

按教师列出：
- 教师
- 课位包
- 课程班
- 学生数

#### 7. 说明

写清：
- 输入文件
- 去重规则
- 求解方式
- 重要优先级
- 已知异常

### 教师简洁版

建议包含：
- `课位包总表`
- `学生课位一览`
- `课程班名单`
- `教师课位一览`

## 三、沟通要求

1. 先说你正在核对什么，再动手。
2. 发现异常后先确认，不要盲猜。
3. 求解完成后先说结果是否可行，再说关键变化。
4. 如果为了减少总调整而动到了高优先级科目，要明确提醒用户。
